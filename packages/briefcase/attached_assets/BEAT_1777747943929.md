## 1. Project Proposal

### 1.1 Vision Statement

Beat puts a multi-agent investigation team in every detective's pocket. It captures evidence from the camera, ingests audio and notes from the phone, and runs scene-tagging, witness-mapping, suspect-background and statement-drafting agents in parallel — visible the entire time — so a detective in the field can leave a scene with a clean, source-grounded case file already started, in any country, in any language.

### 1.2 Problem Statement

| Dimension | Reality |
|---|---|
| **Form factor** | Patrol/detective AI tools in May 2026 (Axon Draft One, Truleo, Tranquility AI TimePilot) are web/desktop bound to records-management systems. Detectives in the field have nothing on the phone. |
| **Audience gap** | Truleo, Tranquility, and Axon target large agencies (typically 500+ officers) on six-figure procurement contracts. Mid- and small-department detectives have nothing they can self-serve. |
| **Cognitive load at the scene** | A detective at a scene is making capture, custody, and follow-up decisions simultaneously. There is no tool that does scene-side cognitive support without yanking them back to a desk. |
| **Evidence handling** | Photos go into a phone roll, notes into a Notes app, audio gets lost. Re-assembly into a case file happens hours later, often with omissions. |
| **Geographic gap** | Even the BigLaw-of-policing tools are US-and-UK-centric. There is no global, jurisdiction-agnostic field assistant. |
| **Trust gap** | Anything used in an investigation must be defensible in court. Most current AI is opaque about how it produced its output, which is a non-starter for evidentiary work. |

### 1.3 Target Users

| Segment | Description |
|---|---|
| **Primary** | Detectives in small-to-mid police departments (under ~500 sworn) and plainclothes investigators worldwide who currently have **no** AI field tooling. |
| **Persona — Detective Reyes** | 41, robbery-homicide, mid-size US department. iPhone + Ford Explorer. Walks into scenes at any hour. Writes incident reports late at night. |
| **Persona — DI Patel** | 36, CID, Greater Manchester Police. PACE/CPIA-aware. Needs OSINT pre-checks before a knock. |
| **Persona — Inspector Souza** | 44, Polícia Civil, Rio de Janeiro. Works in Portuguese; needs witness statements transcribed quickly from Portuguese audio. |
| **Persona — Sub-Inspector Singh** | 32, Karnataka State Police. BNSS 2023-aware. Needs panchanama-style statement drafting from voice notes. |

> **Tier note (locked).** Beat ships with a **free individual tier** marked "not for evidentiary use" and an **agency tier** that requires jurisdiction-appropriate compliance attestation (CJIS in the US, equivalents elsewhere — see §11). Both tiers run the same agents on the same backend.

### 1.4 Goals & Non-Goals

**Goals**

1. Let a detective leave a scene with a clean, structured, source-grounded case file already started.
2. Eliminate "did I check that?" follow-up gaps via the SuspectBackground and WitnessMapper agents prompting next steps in real time.
3. Make every output traceable to either a captured artifact (photo / audio / note / scanned document) or a verified live source.
4. Work the same in any country, in any language, with no user-side configuration.
5. Show the user what the AI is doing, in plain sight, the entire time.

**Non-Goals**

1. Beat does not surveil. It does not listen passively to ambient audio. It records only when the user explicitly hits Record.
2. Beat does not run live face-recognition against any database, public or proprietary.
3. Beat does not auto-submit reports to records-management systems. The detective always reviews and submits.
4. Beat does not replace the chain of custody. It supports it; it does not own it.
5. Beat does not predict guilt, sentence, or recidivism.
6. Beat does not access opposing-party systems. It works only with what the detective captures or imports.
7. Beat does not maintain its own legal/criminal database. It grounds in captured artifacts and verified live sources.

### 1.5 Value Proposition

> "From scene to draft incident report in the time it takes to walk back to the car — visible agents, source-grounded outputs, no contracts to sign."

### 1.6 Success Metrics (post-launch, not buildathon-blocking)

| Metric | Target |
|---|---|
| Time-from-arrival-to-first-draft-statement | ≤ 5 minutes median for a routine scene |
| Capture loss rate | ≤ 1% of photos and audio sessions ever lost in transit |
| Citation verification rate (where applicable) | ≥ 95% of cited public-record references pass the verifier |
| Daily active use per detective | ≥ 1 scene per active workday |
| Country coverage | App functions for users in ≥ 30 jurisdictions in beta |

### 1.7 Scope of This Document

This document fully specifies Beat. It includes shared infrastructure (the JusticeOS backend engine, the agent protocol, the database schema, the deployment design) wherever Beat depends on it, so the document is self-contained. The same backend also serves Briefcase; that overlap is documented identically in `BRIEFCASE.md` so neither document requires the other to be implemented end-to-end.

### 1.8 Stakeholders

| Stakeholder | Interest |
|---|---|
| **End users** (detectives) | Tool that saves time and reduces follow-up gaps without compromising chain-of-custody |
| **Agency leadership** | Auditability, compliance, fit with records-management workflow |
| **Prosecutors / DAs** | Whether AI-assisted reports are defensible at trial |
| **Civil-liberties oversight** | Disclosure of AI use, no surveillance creep, no biometric overreach |
| **Replit Buildathon judges** | Originality, technical depth, impact, fit with Replit's 2026 product story |
| **Future enterprise buyers** (police departments) | Jurisdiction-appropriate attestation, on-prem option, audit trail |

### 1.9 Assumptions

1. Users have a modern smartphone (iOS 17+ / Android 12+) with rear camera and microphone.
2. Users have intermittent internet sufficient to stream agent responses (≥ 1 Mbps).
3. Users are operating within their lawful authority. Beat does not enforce that — it assumes it.
4. For the buildathon demo, a single "Continue as Demo Detective" sign-in is used; production adds Google sign-in plus Replit Auth and an agency-tier attestation flow.
5. Replit's AI integrations (Anthropic, Gemini, OpenAI) and Replit Postgres are available for the build.

## 2. Feasibility Study

### 2.1 Technical Feasibility — Verdict: **HIGH**

| Factor | Assessment |
|---|---|
| **Multi-agent orchestration on TypeScript** | Mastra is production-ready as of May 2026; same framework as Replit Agent 3. Vercel AI SDK 5 handles streaming + tool-call protocol. Risk: low. |
| **Voice-to-investigation** | iOS `Speech` framework + Android `SpeechRecognizer` cover most languages with on-device or cloud fallback. `expo-audio` records WAV/MP4 reliably. Whisper in E2B handles high-quality transcripts when needed. Risk: low. |
| **Camera-to-evidence pipeline** | `expo-camera` + Gemini 3.x vision tagging works on first call (sub-second). EXIF + GPS metadata captured natively. Risk: low. |
| **OSINT for WitnessMapper / SuspectBackground** | Tavily for open-web search, biased to public-records / news domains per §10. No paywalled scraping. Risk: low. |
| **Streaming agent UI on mobile** | Same stack as Briefcase: Vercel AI Elements + AI SDK 5 + Reanimated 4 + Skia at 120 fps. Risk: low. |
| **Field-grade outdoor visibility** | Higher-contrast theme, larger touch targets, max-brightness honor. Risk: low. |
| **Multi-language input/output** | Claude Sonnet 4.6 + Gemini 3.x handle 100+ languages. ASR coverage is the only constraint and we degrade gracefully (text input always available). Risk: low. |

### 2.2 Operational Feasibility — Verdict: **HIGH**

The user workflow (open app → speak the scene → capture photos → watch agents work → walk back to car → review draft) maps cleanly onto a detective's existing field rhythm. Capture-first design (camera FAB and voice prompt are both reachable in one tap from any screen) eliminates context switches.

### 2.3 Legal & Regulatory Feasibility — Verdict: **HIGH (with explicit disclosures and tiering)**

| Concern | Beat's posture |
|---|---|
| **Recording-consent laws (US two-party-consent states, UK RIPA, EU)** | Beat records only on explicit user action. The user is responsible for jurisdictional compliance. UI shows a recording indicator and a per-jurisdiction reminder banner before recording. |
| **CJIS Security Policy (US agency tier)** | Free tier marked "not for evidentiary use." Agency tier requires CJIS attestation and aligns with the cloud-controlled-access model that Truleo and Axon Draft One operate under. |
| **PACE Code C, CPIA disclosure (UK)** | UI surfaces a CPIA disclosure reminder when finalizing a draft incident report; user attests "all unused material has been recorded for disclosure review." |
| **GDPR / UK GDPR (EU/UK)** | Sub-processors disclosed in Settings. Data-handling page explains retention. |
| **DPDP Act 2023 (India)** | Notice + sub-processors disclosed. |
| **LGPD (Brazil)** | Notice + sub-processors disclosed. |
| **EU AI Act high-risk categories (Annex III)** | "Law enforcement risk-assessment of natural persons" is high-risk. Beat **does not** produce risk scores, predictive ranking, or biometric categorization. Beat produces evidence-organization and source-grounded drafts. We document this scope decision visibly. |
| **No live face-recognition** | Hardcoded; no toggle. |
| **Brady / disclosure (US)** | The on-stage citation verifier and full event log (`run_events`) constitute an audit trail of what the AI considered and what it dropped — supportive of disclosure. |

### 3.1 Introduction

Specify the complete behavioral and quality requirements for Beat such that an implementer can build the system from this document without further design decisions.

#### 3.1.2 Scope
Beat is a native mobile application (iOS + Android) plus a shared backend service (the JusticeOS engine) plus integrations to E2B (sandbox), to AI model providers (via Replit AI integrations), to Tavily (web search), and optionally to Google Drive (via MCP) for officers who store case files there.

#### 3.1.3 Definitions, Acronyms, and Abbreviations
See §17 (Glossary).

#### 3.1.4 References

#### 3.1.5 Overview

### 3.2 Overall Description

#### 3.2.1 Product Perspective
Beat is a **client of** the JusticeOS engine. The engine is a server-side TypeScript application using Hono, Mastra, Vercel AI SDK 5, and an OpenAPI-defined contract. Beat consumes that contract through generated React Query hooks. The engine integrates with Replit AI integrations, E2B, Tavily, and (optionally) Google Workspace MCP. Persistence in Replit Postgres with pgvector.

#### 3.2.2 Product Functions (high level)
1. Create and manage investigations.
2. Capture evidence: photos (via camera with auto-tagging), audio (witness/dictation), text notes, document scans, optional Drive imports.
3. Auto-detect jurisdiction and language from captured artifacts and the user's spoken prompt.
4. Run a planner agent that decomposes the investigation goal into a detective role pack.
5. Run subagents in parallel: SceneCaptureTagger, WitnessMapper, SuspectBackground, StatementDrafter.
6. Stream subagent reasoning, tool calls, and partial results to the mobile client over SSE.
7. Render a 4-pane visible agent dashboard with synchronized live updates.
8. Verify every public-records reference via an on-stage verifier in E2B before display.
9. Produce a draft incident report with inline citation chips for review and edit.
10. Persist investigation state, run history, and per-subagent artifacts.

#### 3.2.3 User Characteristics
| Characteristic | Description |
|---|---|
| Technical skill | Low to moderate. Comfortable with mobile photo, voice memos, and basic apps. Not assumed comfortable with shells, APIs, or developer tools. |
| Operating environment | Outdoors and indoors, day and night, often with one hand free; sometimes wearing gloves; sometimes in vehicle. |
| Devices | Modern smartphone (iOS 17+ / Android 12+). Rear camera and mic required. |
| Languages | Any language the LLMs support (100+). UI ships English; localized UI is post-launch. |
| Connectivity | Often spotty or intermittent — graceful degradation required. |

#### 3.2.4 Constraints
| Constraint | Source |
|---|---|
| iOS minimum: 17.0 | Skia + New Architecture |
| Android minimum: API 31 (Android 12) | New Arch + ML Kit |
| Backend runtime: Node 20 LTS | pnpm-workspace template baseline |
| OpenAPI contract is single source of truth | Codegen for client and server |
| No passive audio capture | Recording-consent laws + civil-liberties posture |
| No live face-recognition | Civil-liberties posture (hardcoded off) |
| No predictive risk scoring | EU AI Act + civil-liberties posture |
| All public-record citations shown to user must pass the verifier | Anti-hallucination guarantee |

#### 3.2.5 Assumptions and Dependencies
- Replit Postgres availability with `pgvector` extension installed.
- Replit AI integrations (Anthropic, Gemini, OpenAI) configured.
- Secrets present: `E2B_API_KEY`, `TAVILY_API_KEY`, `GOOGLE_OAUTH_CLIENT_ID_IOS`, `GOOGLE_OAUTH_CLIENT_ID_ANDROID`, `GOOGLE_OAUTH_CLIENT_ID_WEB`, `GOOGLE_OAUTH_CLIENT_SECRET`, `SESSION_SECRET`.
- EAS Build account for iOS/Android dev clients (Expo Go cannot run native modules required: camera, doc scanner, Skia, Google sign-in).

### 3.3 Functional Requirements

Format: **FR-XXX** — Brief title — Description — Acceptance criteria.

#### Onboarding & Auth
- **FR-001 — Demo sign-in.** App presents a single "Continue as Demo Detective" button on first launch. Sets `userId = demo_user_detective` via `x-demo-user` header. *Acceptance:* user reaches Investigations home in ≤ 2 taps.
- **FR-002 — Free vs. Agency tier disclosure.** A first-run sheet explains the tiers. The tier choice sets `users.tier = 'free' | 'agency'` and shows the appropriate compliance banner. *Acceptance:* free tier shows "Not for evidentiary use" everywhere; agency tier requires a CJIS / equivalent attestation step (post-buildathon).
- **FR-003 — Google sign-in (post-demo).** Optional Google sign-in for officers who store case files in Drive.

#### Investigation Lifecycle
- **FR-010 — Create investigation by voice.** From Investigations home, tap "+ New Investigation" → voice prompt: "Tell me what you're working." User dictates; speech-to-text fills the goal. *Acceptance:* an `investigations` row is created with `userId`, `rolePack='detective'`, `goal`, `language` set from the dictation.
- **FR-011 — Create investigation by typing.** Alternative entry: type the goal into a text field. *Acceptance:* same DB result.
- **FR-012 — List investigations.** Investigations home shows all investigations for the current user, sorted most-recently-updated first. *Acceptance:* paginates 25 per page; 60 fps scroll.
- **FR-013 — View investigation.** Tapping an investigation opens the 4-pane Beat dashboard. *Acceptance:* dashboard renders within 300 ms with cached data, then live-updates if a run is active.
- **FR-014 — Rename / archive investigation.** Header menu offers rename and archive. Archive is soft-delete (`deletedAt`). *Acceptance:* archived disappears from main list; visible in Archived view.

#### Capture
- **FR-020 — Photo capture with auto-tag.** Camera FAB opens `expo-camera`. Each photo is uploaded with EXIF + GPS, sent through SceneCaptureTagger (Gemini vision), and a per-photo tag returned (e.g. "front door, no forced entry visible"). *Acceptance:* tag arrives ≤ 4 s after shutter; appears under the photo thumbnail.
- **FR-021 — Audio recording for witness statement.** "Add witness statement" opens a recorder. User taps Record / Stop (no passive listening). On stop, audio is uploaded and transcribed (on-device or via Whisper-in-E2B). *Acceptance:* a 30 s recording yields a structured transcript within 8 s.
- **FR-022 — Document scan.** Document scanner ingests evidence (e.g. receipts, reports) with on-device OCR. *Acceptance:* a 4-page scan completes in ≤ 15 s.
- **FR-023 — Text note.** Quick note pad attached to investigation. *Acceptance:* note saved offline; syncs when online.
- **FR-024 — Drive import (optional).** Officers who store related materials in Drive can import them via the Workspace MCP path (same as Briefcase FR-020). *Acceptance:* shared backend route reused.
- **FR-025 — Capture-anywhere.** A persistent Capture tab gives one-tap access to camera / audio / note from any screen — not gated to a specific investigation. Captured artifacts can be assigned to an investigation later. *Acceptance:* unassigned captures appear in a "Recent captures" tray.
- **FR-026 — Idempotent ingest.** Duplicate artifacts (same SHA-256) are not re-stored. *Acceptance:* contract test verifies.
- **FR-027 — Language detection.** Each captured artifact gets `detectedLanguage`. *Acceptance:* coverage of CLDR top-50 languages for text artifacts.

#### Agent Run
- **FR-030 — Start a run.** User can start a run on an investigation. Server creates a `runs` row, returns `runId`. *Acceptance:* `POST /v1/cases/:id/run` returns ≤ 200 ms.
- **FR-031 — Stream events.** Same protocol as Briefcase (`/v1/runs/:runId/events` SSE). *Acceptance:* first event ≤ 2 s after run start; events monotonic.
- **FR-032 — JurisdictionDetector first.** Same universal subagent. Reads either spoken prompt transcript or earliest captured artifacts. *Acceptance:* `subagent_started` for JurisdictionDetector before any other.
- **FR-033 — Parallel subagent execution.** SceneCaptureTagger, WitnessMapper, SuspectBackground, StatementDrafter run in parallel after JurisdictionDetector completes. *Acceptance:* all four `subagent_started` events fire within 500 ms of each other.
- **FR-034 — Cancel a run.** Same cancel semantics as Briefcase. *Acceptance:* `done` with `cancelled: true` ≤ 2 s after request.
- **FR-035 — Replay a run.** Same replay semantics. *Acceptance:* renders identically to live, at 4× speed.
- **FR-036 — Concurrent runs guarded.** A single investigation may have only one active run. *Acceptance:* second start returns 409.

#### Result Surfacing
- **FR-040 — Scene tags view.** Renders `SceneTags` artifact: per-photo tag, GPS, timestamp, observations. *Acceptance:* tap on a photo shows full-size with tags overlay.
- **FR-041 — Witnesses view.** Renders `WitnessMap`: witnesses list with name, role, statement excerpt, and a small map showing nearby points of interest (Ring camera coverage estimate, public CCTV, neighbors). *Acceptance:* tap on a witness shows full transcript.
- **FR-042 — Suspect background view.** Renders `SuspectProfile`: description, prior public-record incidents within configurable radius/time, sources. *Acceptance:* every prior incident has a tap-through verified URL.
- **FR-043 — Draft incident report.** Renders `IncidentReport`: structured markdown with inline citation chips referencing photos, audio, and OSINT. *Acceptance:* user can edit inline; edits persist.
- **FR-044 — Source viewer.** Tapping any citation chip opens the source — photo / audio / OSINT URL — with the cited span highlighted (where applicable). *Acceptance:* citation always opens to the cited artifact, not a generic detail view.

#### Settings & Compliance
- **FR-050 — Tier banner.** Settings shows current tier (free / agency) and the appropriate compliance attestation status. *Acceptance:* free tier explicitly states "Not for evidentiary use" on every export.
- **FR-051 — Jurisdiction banner.** Settings shows the auto-detected jurisdiction for the current investigation (or aggregated across user's investigations) and dynamically generates a one-paragraph compliance note. *Acceptance:* never shows "unknown" — JurisdictionDetector provides confidence and fallback.
- **FR-052 — Sub-processors disclosure.** Same as Briefcase FR-051.
- **FR-053 — Recording reminder.** Per-jurisdiction recording-consent reminder shown on first audio recording per investigation. *Acceptance:* dismissible; remembered for the investigation; re-prompted if jurisdiction changes.
- **FR-054 — Disclosure / Brady support (post-launch).** Export option produces an audit pack: every event in `run_events`, every artifact, every citation status. *Acceptance:* export is a single .zip with manifest.
- **FR-055 — Recording indicator.** A persistent recording-active indicator is visible whenever microphone is live. *Acceptance:* matches OS recording indicator behavior.

#### Differentiating / Wow-Factor Features (Detective)

- **FR-060 — AR Scene Reconstruction.** Detective slow-pans the camera around a scene; on-device ARKit / ARCore depth + photogrammetry produces a navigable 3D scene mesh with marked evidence anchors. Each anchor links to the photo + voice note captured at that location. *Acceptance:* the resulting `ar_scenes` row stores anchor list with x/y/z + linked `case_files.id`; user can rotate the scene at ≥ 30 fps on iPhone 13 / Pixel 7; export as USDZ for iOS Quick Look (GLTF fallback on Android).
- **FR-061 — Bias Auditor.** A subagent (`BiasAuditor`) reads the detective's own narration, notes, and witness-interview transcripts and flags markers of confirmation bias, premature suspect-locking, or leading-question patterns. Findings render in a dedicated pane with quote, bias category, and a non-judgmental rephrasing suggestion. *Acceptance:* given a fixture transcript with planted leading-question markers, ≥ 80% of plants are flagged; auditor never blocks the run, never speaks publicly — findings are private to the detective.
- **FR-062 — Connected Case Graph.** A subagent (`CaseGraphLinker`) extracts entities (people, vehicles, locations, MO descriptors) from this investigation and queries previously-ingested investigations belonging to the same agency tenant for shared edges. Result renders as a force-directed graph (`react-native-skia`) with this case at center and connected cases as satellite nodes. *Acceptance:* tapping any node opens that case's summary; edge labels show the shared entity; graph requires ≥ 2 prior investigations to render meaningfully (otherwise empty state).
- **FR-063 — Composite Sketch from Voice.** Witness describes suspect verbally; transcription feeds `CompositeArtist` which calls Imagen / Gemini image generation iteratively. Witness verbal corrections regenerate the image until accepted. *Acceptance:* first-pass sketch renders ≤ 12 s after end of initial description; each "more / less / change X" correction triggers a regeneration in ≤ 10 s; final image saved as a `case_files` row with `sourceType='composite'` and the full description thread persisted; image carries an "AI-generated suspect sketch — not a photograph; not for biometric matching" watermark.
- **FR-064 — Body-cam Highlight Reel.** Detective drops a long body-cam video; `BodycamEditor` scans frames for relevance (interactions, evidence in frame, key statements via aligned audio) and produces a highlight reel with chapter markers. *Acceptance:* a 4-hour input produces a ≤ 10-min highlight in ≤ 5 min server time on the demo tier; chapters are tappable; original footage is preserved untouched and a chain-of-custody event is recorded for both source and derivative.
- **FR-065 — Hands-free Crime Scene Mode.** A wake-word session ("Reyes…") enables full voice control: "photo this", "tag as weapon", "note: blood spatter three feet from victim", "next witness". On-device wake-word + push-to-talk fallback. *Acceptance:* every voice command maps to a deterministic action with audio + haptic confirmation; mic indicator always visible; auto-disengages after 60 s of silence; wake-word audio never leaves the device.
- **FR-070 — Glass Box Agent Theater.** Same contract as Briefcase FR-070 — full agent DAG with token-by-token streaming inside each tool leaf. *Acceptance:* a single screen renders the full DAG of a run within a 16 ms frame budget under load; replay mode reproduces the tree identically.
- **FR-071 — Time-Travel Agent Debugger.** Same contract as Briefcase FR-071 — rewind / fork / replay any run with edited inputs. Branches stored in `run_branches`. *Acceptance:* a forked branch executes and streams its own events; the original run remains unchanged; both branches are diffable side by side.
- **FR-072 — Verifiable Citations Everywhere.** Same contract as Briefcase FR-072 — every claim carries a `Citation` and unverified ones are dropped before render. *Acceptance:* contract test asserts every artifact field of type `claim` carries at least one verified citation; UI never renders an unverified claim.

### 3.4 Non-Functional Requirements

#### Performance
- **NFR-P-001 — Cold start.** App opens to home in ≤ 1.5 s on iPhone 13 / Pixel 7.
- **NFR-P-002 — Photo-to-tag latency.** ≤ 4 s P50.
- **NFR-P-003 — Voice-to-text latency.** ≤ 1 s P50 for short prompts (≤ 10 s).
- **NFR-P-004 — TTFB on SSE stream.** ≤ 2 s.
- **NFR-P-005 — End-to-end run latency.** P50 ≤ 90 s for a typical scene (8 photos, 1 voice prompt, 1 audio statement).
- **NFR-P-006 — Frame rate.** ≥ 60 fps under load on iPhone 13 / Pixel 7.

#### Reliability
- **NFR-R-001 — Stream resumption.** SSE auto-reconnect with `since=<lastIndex>`.
- **NFR-R-002 — Idempotent run start.** Same as Briefcase NFR-R-002.
- **NFR-R-003 — Graceful agent failure.** A failing subagent emits `error`; others continue; pane shows error + retry.
- **NFR-R-004 — Capture durability.** Captured artifacts queue locally if offline; sync on reconnect; never silently dropped.
- **NFR-R-005 — Backend uptime target.** ≥ 99.5%.

#### Security
- **NFR-S-001 — Transport.** TLS 1.3 only.
- **NFR-S-002 — Secret storage.** Refresh tokens AES-256-GCM keyed from `SESSION_SECRET` HKDF.
- **NFR-S-003 — Sandbox isolation.** Every E2B execution in a fresh Firecracker microVM.
- **NFR-S-004 — Recording indicator.** OS recording indicator always present during recording (no hidden capture).
- **NFR-S-005 — No biometric matching.** Hardcoded off. No face-recognition tools wired in.
- **NFR-S-006 — Local artifact storage.** Captured artifacts encrypted at rest on device using `expo-secure-store`-backed key.
- **NFR-S-007 — PII handling.** Disclosed in §11; agency tier supports per-tenant zero-retention.

#### Usability (field-grade)
- **NFR-U-001 — One-handed reachable.** All primary actions reachable in the bottom 60% of the screen (capture FAB, agent panes, voice prompt).
- **NFR-U-002 — Outdoor visibility.** Higher contrast theme; brightness boost on capture.
- **NFR-U-003 — Glove-friendly touch targets.** Minimum 56 dp.
- **NFR-U-004 — Reduced motion.** Honors `prefers-reduced-motion`.
- **NFR-U-005 — Haptics.** Discrete haptics on capture, agent state changes, recording start/stop.
- **NFR-U-006 — Voice-first entry.** Voice prompt is the *default* on the New Investigation screen. Typing is a one-tap alternative.
- **NFR-U-007 — Battery awareness.** Skia halos slow to 4 s loop when battery < 20%.

#### Usability — Beginner & Onboarding
- **NFR-U-008 — First-launch boot sequence.** App boots with a particle-assemble logo and persona introduction (Reyes literally introduces himself: *"Reyes here. I'm your field partner."*). Skippable; runs once per install (`appState.firstRunCompleted`).
- **NFR-U-009 — Practice investigation.** Empty home offers a one-tap "Try a sample investigation" using local fixture data; runs entirely against canned events; never hits backend.
- **NFR-U-010 — Spotlight tour.** First run renders pulsing rings around the next action (mic → first capture → tag → witness map → draft). Voice-over by Reyes. Skippable; repeatable from Settings → "Replay tour".
- **NFR-U-011 — Plain-English mode.** Header toggle replaces forensic jargon (e.g. "chain of custody", "BNSS", "PACE Code C", "spoliation") with plain-language equivalents; glossary in `lib/i18n/glossary.{lang}.json`.
- **NFR-U-012 — Contextual helper.** Floating "What can I do here?" pill on every screen reveals 3 contextual actions in plain language ("Capture a photo", "Record a statement", "Open the witness map").
- **NFR-U-013 — Plain-language confidence.** Numeric confidences render as labels (`Pretty sure` ≥ 0.85, `Likely` ≥ 0.6, `Worth checking` < 0.6); numeric value still available on long-press.
- **NFR-U-014 — Universal Undo.** Every destructive action emits a 5-second `<UndoToast>`; session-scoped Recent Actions in Settings allows long-undo within session.

#### Usability — Modern Visual Language
- **NFR-U-015 — Liquid Glass everywhere.** Every floating surface uses native iOS 26 / Android 16 glass material; never a plain solid sheet. (Slightly more opaque on Beat for outdoor visibility — see §7.1.)
- **NFR-U-016 — Aurora background.** Every primary screen renders a Skia mesh-gradient aurora that reacts to agent state (calm idle / rippling working / pulse on completion) via `AmbientReactor` (§5.7). Tactical-green undertone.
- **NFR-U-017 — OLED true-black + dynamic accent.** Background `#000000` on OLED devices; primary accent oscillates ±8% lightness with agent activity.
- **NFR-U-018 — Variable typography.** Variable-axis fonts (Inter Variable + Geist Variable) animate weight on emphasis, focus, and active states.
- **NFR-U-019 — 60 fps under load.** All Skia futurism (aurora, halos, neural tree, particles, holographic shimmer, AR overlays) runs on UI thread via Reanimated worklets and stays at 60 fps on iPhone 13 / Pixel 7. Auto-throttles to 30 fps when battery < 20%.
- **NFR-U-020 — Holographic AI marker.** AI-generated content (sketches, drafts, tags) carries a subtle iridescent gradient border so authorship is *felt*, not just labeled.
- **NFR-U-021 — Sound design (opt-in).** 12-clip micro-sound bank (≤ 80 ms each) for taps, agent spawn, agent done, citation stamp, success, errors, wake-word ack, capture shutter. Off by default; user opt-in in Settings.

#### Engine — Performance / Cost / Trust
- **NFR-E-001 — Hybrid model routing.** Every model call is routed through `ModelRouter` (§5.7); the chosen model and rationale are recorded with cost on `model_routing_decisions`.
- **NFR-E-002 — Speculative execution.** Subagents may speculatively start before the planner finalizes its plan; outputs from discarded branches are persisted to `speculative_branches` but not rendered.
- **NFR-E-003 — Semantic cache hit ratio.** ≥ 30% of LLM calls on a typical replayed run hit the semantic cache; cache key is `{role, promptEmbedding, toolArgsHash}`.
- **NFR-E-004 — Streaming tool calls.** Every tool emits intermediate progress events (≥ 1 every 2 s for any operation > 4 s) so the Glass Box stays alive.
- **NFR-E-005 — Self-healing retries.** Subagent failures auto-retry up to 2× with smaller scope or alternate model; failures past the retry budget surface as `error` events with the reformulation history attached.
- **NFR-E-006 — Live cost transparency.** The Glass Box header always displays running cost-so-far in USD with model breakdown; final cost persisted on `agent_costs.totalUsd`.
- **NFR-E-007 — Constitutional gate.** Every artifact passes a values check before render; outputs violating role-pack constraints (no biometric matching, no risk-scoring of persons, free-tier export-only restrictions) are dropped and recorded in `policy_drops`.
- **NFR-E-008 — Audit bundle.** Every completed investigation produces a signed audit bundle (events + artifacts + citations + prompt versions + costs + model decisions + chain-of-custody manifest) downloadable via `/v1/runs/:runId/audit`.
- **NFR-E-009 — OpenTelemetry tracing.** Every agent call, tool call, and model call gets a span; traces queryable for the Glass Box and exportable to any OTel-compatible backend.
- **NFR-E-010 — Quality judge.** A small evaluator LLM scores each subagent output against the role-pack rubric; outputs below threshold are auto-retried once before render.
- **NFR-E-011 — Cost guardrails.** Per-user / per-tenant cost ceilings; runs nearing the ceiling automatically degrade to cheaper models; exceeding the ceiling halts the run with a clear in-UI message.
- **NFR-E-012 — Prompt versioning.** Every prompt is versioned and the version recorded on each run so outputs are reproducible; A/B variants supported via `prompt_versions`.
- **NFR-E-013 — Hierarchical memory.** Three tiers: working (per-run scratchpad), case (per-investigation via pgvector), tenant (per-user / per-agency). Retrieval is automatic by relevance; tenant scope is RLS-strict.
- **NFR-E-014 — Inter-agent collaboration.** Subagents may post messages to each other through `agentMessageBus`; messages persisted to `agent_messages` and visible in the Glass Box.
- **NFR-E-015 — Multi-strategy citation verification.** Citations pass through `verifierBank` which selects substring / semantic / temporal-anchor / image-cropbox / GPS-radius strategy by source type; failures drop the citation silently.
- **NFR-E-016 — Replay regression.** Every demo-quality run is auto-saved to `replay_cases`; CI replays the suite and asserts identical or improved outputs against goldens.
- **NFR-E-017 — Chain-of-custody manifest.** Every captured photo / audio / video file is recorded with sha256, capture device, capture timestamp, GPS (when available), and an append-only event log; included in the audit bundle (NFR-E-008).

#### Maintainability
- **NFR-M-001 — Type safety.** Zero `any` in client agent rendering code. Zod-validated at the boundary.
- **NFR-M-002 — Codegen-only DTOs.**
- **NFR-M-003 — Logger discipline.** Same as Briefcase NFR-M-003.

#### Portability
- **NFR-O-001 — Same APK/IPA path.** iOS and Android share the OpenAPI contract.
- **NFR-O-002 — Multi-language ready.** All user-facing strings in `i18n` resource bundle.

#### Compliance
- **NFR-C-001 — Disclosure visibility.** Sub-processors and AI usage disclosure reachable from every screen that initiates an AI call or capture.
- **NFR-C-002 — Citation provenance.** Every public-record citation includes a tap-through URL to the verified source.
- **NFR-C-003 — Tier markings.** "Not for evidentiary use" present on every export, draft preview, and share action under the free tier.
- **NFR-C-004 — No predictive scoring on persons.** Hardcoded; documented; absent from UI.

### 4.2 Use Cases

#### UC-01 — Investigate a scene from voice + photos
**Actor:** Detective. **Trigger:** Tap "+ New Investigation" → voice prompt.
**Preconditions:** App permissions for camera + mic + (optional) GPS granted.
**Main flow:**
1. User dictates the goal: "Robbery at 4422 Ash Street tonight, victim Lisa Park, suspect tall white male, dark hoodie."
2. Speech-to-text fills the goal. App calls `POST /v1/cases` with `rolePack='detective'`.
3. App opens 4-pane dashboard for the investigation.
4. App calls `POST /v1/cases/:id/run`.
5. SSE stream begins.
6. JurisdictionDetector runs; jurisdiction set on the investigation.
7. Four subagents start in parallel.
8. User taps Camera FAB and captures 8 scene photos. Each photo is uploaded; SceneCaptureTagger emits per-photo tags.
9. WitnessMapper begins building the map; SuspectBackground searches public records; StatementDrafter starts streaming a draft.
10. User taps "Add witness statement," records 30 s of audio, gets transcript. WitnessMapper updates.
11. Run completes; draft incident report is ready for review.
**Postconditions:** Investigation has artifacts; run history persisted; user can re-open and review/edit.

Identical to Briefcase, with these Beat-specific additions:

| Failure | Behavior |
|---|---|
| Camera permission denied | Capture FAB shows a prompt to open Settings; user can still proceed via voice / text. |

### 5.7 Cross-Cutting Design

Same as Briefcase §5.7. Beat additionally:
- Renders a persistent recording indicator at the top safe-area whenever the mic is active (`<RecordingIndicator>`).
- Records a per-investigation log line on every capture, every run start, every export — supports auditability under the agency tier.

#### Motion / Sound / Haptic / Ambient Systems (mobile)
- **MotionSystem (`lib/motion`)** — central spring & easing presets. 6 named tokens: `whisper`, `soft`, `bouncy`, `elastic`, `snap`, `dramatic`. All animations resolve through these tokens. Reduced-motion fallback collapses every spring to a 120 ms opacity crossfade.
- **SoundSystem (`lib/sound`)** — 12-clip micro-sound bank (`tap`, `confirm`, `error`, `agentSpawn`, `agentDone`, `citationStamp`, `swipe`, `success`, `wakeWordAck`, `captureShutter`, `recordOn`, `recordOff`). All clips ≤ 80 ms; off by default; user opt-in in Settings (NFR-U-021).
- **HapticSystem (`lib/haptic`)** — 8-pattern taxonomy mapped to event types: `lightClick`, `success`, `warning`, `agentStart`, `agentDone`, `wakeDetected`, `citationVerified`, `recordingOn`. Wraps `expo-haptics` with deterministic semantics.
- **AmbientReactor (`lib/ambient`)** — single subscription to agent state that drives the global Aurora background, tab-bar magnetic indicator, and ambient color-shift. One source of truth so animations never desync.
- **GlassBoxBus (`lib/glassbox`)** — typed event bus that mirrors the SSE stream into the Glass Box neural-tree renderer (A3) without a second subscription.
- **OutdoorMode (`lib/outdoor`)** — single subscription to ambient light sensor (`expo-sensors` `LightSensor`) that boosts contrast, dims aurora, and forces brightness when readings cross outdoor threshold.

## 10. Internationalization & Jurisdiction Handling

Beat is **universal by architecture**. There is no "we don't support your country" anywhere in the app.

### 10.1 How It Works

| Layer | Behavior |
|---|---|
| Capture | Camera, audio, text — all language-agnostic. |
| Voice prompt | iOS / Android speech-to-text supports the device's configured locales; transcript flows into the same agent path as typed input. |
| LLMs | Claude Sonnet 4.6 + Gemini 3.x handle 100+ languages. Output matches input language. |
| **JurisdictionDetector** | First subagent on every run; output saved to `cases.jurisdictionContext`. User never picks. |
| **Curated trusted sources** | Static config (`lib/jurisdictions.ts`) per country; OSINT calls biased to listed domains. |
| **On-stage citation verifier** | Every public-record cite is fetched and quote-matched. Failed cites dropped silently. |
| **Compliance banner** | Settings shows detected jurisdiction + dynamically generated procedural guidance paragraph. |
| **Recording-consent reminder** | Shown before first audio recording per investigation; jurisdiction-aware. |

### 10.2 Curated Trusted Source List (locked, top 15)

Same list as Briefcase §10.2 (CourtListener / BAILII / CanLII / AustLII / Indian Kanoon / etc.). For Beat the bias also includes public news domains permitted in the user's jurisdiction; OSINT is grounded only in publicly available sources.

### 10.3 What This Is NOT

- Not a stored copy of every country's laws or criminal records (impossible and inappropriate).
- Not a curated criminal database we maintain.
- Not "the AI knows the suspect" — every claim is grounded in an artifact the user captured or a verified public source.

### 10.4 Honesty Guarantee

Every claim Beat shows the user has come from a captured artifact (photo, audio, note, scan) **or** from a URL the verifier confirmed at run time. There is no answer the app gives that it cannot point at a source for. Same in Lima, Lagos, Lisbon, or Louisville.

## 11. Security, Privacy & Compliance Design

### 11.1 Threat Model (STRIDE summary)

| Threat | Mitigation |
|---|---|
| **Spoofing** | Demo: header. Production: cookie + tier check. |
| **Tampering** | TLS 1.3. SHA-256 on every captured artifact. |
| **Repudiation** | `run_events` monotonic log; per-investigation log lines for capture / run / export. |
| **Information disclosure** | Row-level filtering by `userId`. Postgres no-superuser role. Sub-processor disclosure. |
| **DoS** | Per-user run rate limit (post-launch). E2B bounded per call. |
| **Elevation of privilege** | Sandbox isolation; explicit, validated tool calls. |
| **Surveillance creep** | No passive audio. No background recording. No biometric matching. Hardcoded. |

### 11.2 Data Classification

| Class | Examples | Handling |
|---|---|---|
| **Highly sensitive** | Captured photos containing faces, audio of witnesses, GPS coordinates of incidents, draft narratives | TLS only; encrypted at rest on device; sub-processors disclosed |
| **Sensitive** | Run events, planner / agent outputs | TLS only; retained 90 days |
| **Internal** | File IDs, hashes, EXIF | TLS only |
| **Public** | App version, model versions | n/a |

### 11.3 Encryption

| At rest | In transit |
|---|---|
| Refresh tokens AES-256-GCM keyed from `SESSION_SECRET` HKDF | TLS 1.3 |
| Captured artifacts encrypted on device using `expo-secure-store`-backed key | TLS to Anthropic / Gemini / OpenAI / E2B / Tavily |
| Postgres at rest encryption (Replit-provided) | |

### 11.4 Sub-processor Disclosure (in-app)

Same list as Briefcase §11.4. Additionally for Beat's free tier, all exports are watermarked and the JSON manifest discloses every sub-processor that touched any artifact in that case.

### 11.5 Compliance Postures by Region

| Region | Posture |
|---|---|
| US (free) | "Not for evidentiary use" mark on every export. |
| US (agency) | CJIS attestation flow (post-buildathon). Sub-processors disclosed. |
| EU / UK | GDPR / UK GDPR. ICO surveillance code awareness. |
| India | DPDP 2023. |
| Brazil | LGPD. |
| All others | Generic "review your jurisdiction's evidence and data-protection rules." |

### 11.6 Civil-Liberties Posture (locked)

- **No predictive risk-scoring of natural persons.** Hardcoded; absent from API and UI.
- **No biometric matching.** Hardcoded; absent from tools.
- **No passive audio.** Recording requires explicit user action.
- **Recording indicator always visible** when mic active.
- **Citation honesty.** Every public-record claim shown to user has been verifier-fetched and quote-matched.

### 11.7 Audit & Forensics

- Every run, capture, edit, and export logged with monotonic ordering.
- Replay possible via `run_events`.
- Export pack includes a full manifest of all artifacts, all agent events, and verifier results — supports disclosure obligations.
